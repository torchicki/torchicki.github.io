4624​

https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-10/security/threat-protection/auditing/event-4624

Logon TypeLogon TitleDescriptionメモ
0SystemUsed only by the System account, for example at system startup.----
2InteractiveA user logged on to this computer.コンピューターのコンソール(入出力装置)を使用したログオン。
3NetworkA user or computer logged on to this computer from the network.ネットワーク経由での共有フォルダやプリンタへのアクセス。
(※多くの場合、IISへのログオンはネットワークログオンとして記録される)
4BatchBatch logon type is used by batch servers, where processes can be run on behalf of a user without their direct intervention.タスクスケジューラーの実施により発生したログオン。
(※タスクスケジューラーサービスによりタスクスケジューラーが開始されたときに記録される)
5ServiceThe Service Control Manager started a service.サービス開始に伴うサービス起動アカウントによるログオン。
7UnlockThis workstation was unlocked.ワークステーションのロックを解除。
8NetworkCleartextA user logged on to this computer from the network. The user's password was passed to the authentication package in its unhashed form. The built-in authentication packages all hash credentials before sending them across the network. The credentials don't traverse the network in plaintext (also called cleartext).ネットワーク経由でクリアテキストの資格情報を使用したログオン。
9NewCredentialsA caller cloned its current token and specified new credentials for outbound connections. The new logon session has the same local identity, but uses different credentials for other network connections.RunAsコマンドに、オプションとして「/netonly」を指定したプログラムの開始
10RemoteInteractiveA user logged on to this computer remotely using Terminal Services or Remote Desktop.RDP接続を使用したログオン。
11CachedInteractiveA user logged on to this computer with network credentials that were stored locally on the computer. The domain controller wasn't contacted to verify the credentials.キャッシュされている認証情報を元に実施したログオン。
(※ドメインコントローラーが利用できないときなどに使用される)
12CachedRemoteInteractiveSame as RemoteInteractive. This type is used for internal auditing.^^
13CachedUnlockWorkstation logon.^^

参考リンク https://blogs.manageengine.jp/needs_of_auditing_4624/

4625​

https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-10/security/threat-protection/auditing/event-4625

Logon Typeに関しては4624イベントを参照。

Statusdescriptionメモ
0XC000005E"There are currently no logon servers available to service the logon request."This issue is typically not a security issue, but it can be an infrastructure or availability issue.---
0xC0000064"User logon with misspelled or bad user account". Especially if you get several of these events in a row, it can be a sign of a user enumeration attack.ユーザー名が間違っているか存在しない
0xC000006A"User logon with misspelled or bad password" for critical accounts or service accounts. Especially watch for a number of such events in a row.ユーザーのパスワードが間違っている
0XC000006D"This is either due to a bad username or authentication information" for critical accounts or service accounts. Especially watch for a number of such events in a row.ユーザー名か認証情報が間違っている
0xC000006F"User logon outside authorized hours".ユーザーが許可されていない時間にログオンを実施
0xC0000070"User logon from unauthorized workstation".ユーザーが許可されていないワークステーションからログオンを実施
0xC0000072"User logon to account disabled by administrator".ユーザーが無効化されている
0XC000015B"The user has not been granted the requested logon type (aka logon right) at this machine".ユーザーが許可されていないログオンタイプを要求
0XC0000192"An attempt was made to logon, but the Netlogon service was not started". This issue is typically not a security issue but it can be an infrastructure or availability issue.---
0xC0000193"User logon with expired account".ユーザーのアカウント有効期限が切れている
0XC0000413"Logon Failure: The machine you are logging onto is protected by an authentication firewall. The specified account is not allowed to authenticate to the machine".---

4720​

ユーザの作成を記録する。
作成元ユーザを確認することで、運用以外の動作かを確認する。

https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-10/security/threat-protection/auditing/event-4720

4723​

パスワード変更試行を記録する。 実行者はアカウントの所有者自身。
新しいパスワードがパスワードポリシーを満たしていない場合等は失敗として記録される。
変更の際にユーザがパスワードを間違えた場合、このイベントは記録されず 4771や4776が記録される。

https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-10/security/threat-protection/auditing/event-4723 https://www.ultimatewindowssecurity.com/securitylog/encyclopedia/event.aspx?eventid=4723

4724​

パスワードのリセットを記録する。
実行者は管理者等のパスワードリセットの対象アカウントとは別のアカウントとなる。 新しいパスワードがパスワードポリシーを満たしていない場合等は失敗として記録される。

https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-10/security/threat-protection/auditing/event-4724 https://www.ultimatewindowssecurity.com/securitylog/encyclopedia/event.aspx?eventid=4724
